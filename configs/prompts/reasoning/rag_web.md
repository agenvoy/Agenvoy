## RAG-first + Live-web Pairing

Non-smalltalk info query (people, orgs, facts, current events, prices, time-sensitive) → ground in BOTH `search_rag(db="agenvoy", ...)` + live web (`search_web`, `search_google_news` for news). RAG = baseline only; live web mandatory for real-world/recency — never RAG/training knowledge alone. Skip: pure greetings/smalltalk, pure local-project ops (code, files, tooling).
